MediScan AI 界面设计分析
一、设计风格：Soft UI Evolution（软 UI 进化版）
这是整个设计的核心。它不是传统的纯扁平设计，也不是对比度偏低的纯拟物新态（Neumorphism），而是两者的折中进化版本：

特征	纯扁平设计	纯拟物新态	Soft UI Evolution（本页）
阴影	无或极淡	多层强烈浮雕阴影	柔和多层阴影，比扁平有层次，比拟物清晰
对比度	高	低（WCAG 不合格）	WCAG AA+，4.5:1 以上
圆角	0–4px	12–16px	8–24px 分层使用
动画	即时或无	偏慢	150–300ms，有弹性曲线
二、配色体系：医疗保健青蓝调
来自 Healthcare App 色板，关键词是 "calm cyan + health green"（沉静青蓝 + 健康绿）：

主色 (Primary)    #0891B2  青蓝色 — 用于主按钮、链接、进度条
辅色 (Secondary)  #22D3EE  浅青 — 用于 hover 态、高亮背景
强调色 (Accent)   #059669  翠绿色 — 用于 CTA、成功状态、训练按钮
背景 (Background) #ECFEFF  极浅青 — 整体页面底色
表面 (Surface)    #FFFFFF  白色 — 卡片、面板
前景 (Foreground) #164E63  深青 — 主文字色
色彩语义分工：

cyan（青蓝）→ 信息类图标背景（ROI 标注、数据集）
green（翠绿）→ 成功/确认类图标背景（标注、预测）
amber（琥珀）→ 需要关注的操作（训练模型按钮）
red（红色）→ 危险/破坏性操作（关闭按钮悬停、删除）
为什么不用常规医疗蓝：#2563EB 偏冷硬，#0891B2 夹带绿色成分，视觉上更温暖、放松，符合"relaxing medical colors"的要求。

三、字体系统
用途	字体	分类	特性
标题	Figtree	Sans-serif	几何感强、现代、圆润友好
正文	Noto Sans	Sans-serif	极高可读性、多语言覆盖、中性专业
选择理由：medical, clean, accessible, professional, healthcare, trustworthy（医疗、干净、无障碍、专业、可信赖）

四、布局结构：Bento Grid（便当盒网格）
采用 Apple 风格的 Bento Grid 布局，用于核心功能卡片区：

┌──────────────────────┬──────────┬──────────┐
│                      │ Labeling │ Dataset  │
│   ROI Annotation     │  卡片    │  Generation│
│   (宽卡 × 高卡)       ├──────────┼──────────┤
│                      │  Train   │ Predict  │
└──────────────────────┴──────────┴──────────┘
与常规等宽网格的差异：

ROI Annotation 占据 2×2 格（最重要功能），更大的视觉权重
其余 4 个功能卡片各占 1×1
卡片大小反映功能重要性，而非均匀分配

六、动画与交互
类型	参数	用途
快速过渡	150ms cubic-bezier(0.4, 0, 0.2, 1)	按钮悬停、开关切换
基础过渡	200ms 同上	卡片悬停、上传区
慢速过渡	300ms 同上	进度条、模态框
弹性曲线	cubic-bezier(0.34, 1.56, 0.64, 1)	Toast 入场、模态框入场
设计原则：

卡片悬停 → translateY(-2px) + 阴影加深（不改变布局尺寸）
按钮按下 → scale(0.97)（轻微缩小反馈）
模态框入场 → scale(0.95→1) + opacity(0→1) 弹簧曲线
Toast 入场 → 从右侧滑入，4 秒后滑出
七、设计系统总结
维度	选择
风格	Soft UI Evolution（柔和阴影 + 高对比度）
配色	Healthcare App（青蓝 + 翠绿 + 极浅青背景）
字体	Figtree（标题）+ Noto Sans（正文）
布局	Bento Grid（不对称卡片网格）
圆角	8–24px 分层使用
阴影	带主色调的多层柔和阴影（非纯黑）
动画	150–300ms，弹性曲线，尊重 prefers-reduced-motion
无障碍	WCAG AA+，跳转链接，ARIA 标签，键盘导航，焦点环
图标	纯 SVG 内联（无 Emoji），统一 2px 描边
